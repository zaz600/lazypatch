Version 0.1b
============
Стартовая версия.


Version 0.2b
============
05-08-2010
- Изменен механизм обработки исключений. Теперь при возникновении исключения в директории с программой
  создается файл "debug.trc", содержащий трейс вызовов. Изменения также коснулись совместимости, теперь поддерживаются
  версии Python начиная с 2.2.
06-08-2010
- Переработан механизм проверки канкарентов отвечающих за перенос в ГК. Теперь перед остановкой appsTier
  для них будет проставлен флаг hold, затем будет осуществлен поиск таких канкарентов в статусе running.
  Если их колчество больше нуля, то программа будет ждать 5 минут, затем проверит количество снова. И так
  12 итераций по 5 минут (итого проверка корректного завершения будет осуществляться в течение часа). Если
  после прохождения всех шагов итерации так и не удалось дождаться завершения канкарентов, то будет осуществлен
  аварийный выход с записью о неудачном патчевании в лог-файл. appsTier не будет остановлен и патчи накатываться
  не будут.
- Добавлен новый флаг: (-i, --nokill). Если указан, то не будут принудительно завершаться никакие процессы.
  В противном случае после штатной остановки appsTier и 15-ти минутного ожидания (если указано --nosleep то сразу же
  после остановки) будут принудительно завершены процессы frmweb (подвисают на AIX) и процессы относящиеся к CM:
  FNDLIBR (ICM и Standard Manager), POXCON (Purchase Order Document Approval) и RCVOLTM (Receiving Transaction Manager).

Version 0.4b
============
08-12-2010
- Доработано завершение процессов на WIN-платформах. Новый мехнизм требует установленного MKS. Убрана зависимость от framedyn.dll.
- Убран флаг check_version=N из команды запуска патчевания. Патчи теперь всегда ставятся с проверкой версии.
- Теперь по-умолчанию перед установкой патчей (после остановки appsTier) будет выполнен перезапуск БД. Для опционального
  отключения этого действия создан флаг (-b, --no-restart-db). В связи с этим в конфигурационный файл добавлен новый
  параметр syspwd (пароль пользователя sys). Для успешного перезапуска базы, в ifile должен быть выставлен параметр
  remote_login_passwordfile=exclusive а также в $ORACLE_HOME/dbs/ создан файл orapw<SID_NAME> (например orapwATESTE). Создать
  его можно командой "orapwd file=$ORACLE_HOME/dbs/orapwATESTE password=<sys password>". Подробнее см. в документации.
  Если рестартовать БД не удалось, будет осуществлен аварийный выход, патчи устанавливаться не будут, appsTier останется остановленным.
- Останов и проверка канкарентов переноса в ГК теперь осуществляется по-умолчанию. Ключ -q теперь имеет противоположное значение,
  т.е. если он указан, проверка осуществляться не будет. Проверьте свои записи в crontab!

Version 0.5b
============
17-01-2011
- Изменен механизм проверки на запуск более одного экземпляра программы. Вместо подсчета запущеных процессов себя
  (который на разных ОС может отрабатывать по-разному) теперь используется lock-файл, который создается в
  каталоге со скриптом при запуске.
- Добавлен флаг (-K, --kill-all) для убиения ВСЕХ cессий пользователя APPS на уровне БД, а затем и на уровне ОС
  перед установкой каждого патча. Для корректной работы этой функции требуется настроеная беспарольная
  SSH-авторизация со стороны appsTier на dbTier а также должна быть возможность подключиться к БД как sysdba с appsTier.
  Подробнее см. в документации. Алгоритм работы таков:
  берутся идентификаторы всех сессий пальзователя APPS; затем для каждой такой сессий выполняется alter system kill session;
  после этого на dbTier отправляется команда вида ssh tdboebs@aXX00y03.ufkXX.roskazna.local "kill -9 123 456 789" с PIDами сессий.
  Затем начинается установка патча.
20-01-2011
- На WIN-платформе теперь вместо xxt_patch.cmd вызывается xxt_patch_t.cmd, где переменные, определяющие логин/пароль APPS имеют
  фиксированное значение.
